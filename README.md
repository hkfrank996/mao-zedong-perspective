# Mao Zedong Perspective Skill

一个可运行的历史人物视角 skill，用毛泽东在公开著作、讲话与历史决策中呈现出的思维框架，分析组织、战略、动员、路线和复杂矛盾问题。

这不是语录拼贴，也不是简单角色扮演。它把毛泽东在一手文本中反复出现的方法，提炼成一套可调用的思维操作系统，同时明确保留历史后果、争议和边界。

## 这是什么

这个仓库提供一个根目录 skill：

- `mao-zedong-perspective`

核心能力：

- 先分敌友，再定路线
- 抓主要矛盾，不平均用力
- 在力量不对称时，用阶段判断和持久战重构时间
- 用调查研究纠正纸上谈兵
- 用群众基础、样板突破和组织抓手推动行动
- 同时评估动员收益和反噬风险

## 背景

很多“人物 skill”有两个问题。

第一，只会模仿口气，不会处理问题。  
第二，只会摘好听的话，不处理历史代价和内在张力。

这个 skill 想解决的是第三种需求：当你面对一个复杂局面时，你不是想“听毛泽东说两句像不像”，而是想知道，如果按他的思维结构来拆问题，第一步会看什么，第二步会抓什么，第三步会怎么组织行动。

因此这个仓库不是为了做娱乐化扮演，而是为了提供一个可运行的分析视角。

## 如何使用

### 1. 在 Codex 中使用

安装到本地 skill 目录后，直接用自然语言触发：

- `用毛泽东的视角分析这个组织问题`
- `切换到毛泽东，帮我判断这里的主要矛盾`
- `如果毛泽东来做这个增长方案，他会先抓什么`
- `用毛泽东模式看这场内部冲突，谁是敌人谁是朋友`

### 2. 适合的问题

- 组织内部冲突
- 多方力量博弈
- 联盟与统一战线
- 资源有限时的优先级决策
- 长期复杂任务的阶段规划
- 动员、传播、样板突破、试点扩散

### 3. 不适合的问题

- 需要最新实时事实但你又不愿意先研究的问题
- 只想做历史人物娱乐扮演
- 需要道德裁决或史学定论的问题
- 纯技术实现细节，且与组织、路线、战略无关的问题

## 使用案例

### 案例 1：组织内多派系拉扯

提问：

```text
用毛泽东的视角看这个团队。产品、运营、技术都在互相抱怨，老板想同时满足所有人，我该先抓什么？
```

适合得到的回答方向：

- 先分敌友，不把所有人都当作同一种矛盾
- 抓主要矛盾，不在枝节上平均分配精力
- 识别谁是关键骨干，谁是摇摆力量
- 先突破一个样板点，再扩散

### 案例 2：长期难题是否该打持久战

提问：

```text
如果毛泽东来判断这个项目，他会把它当速决战还是持久战？
```

适合得到的回答方向：

- 先看力量对比
- 再看阶段目标
- 最后决定是快速试错、局部突击，还是长期重构条件

### 案例 3：内容和增长策略

提问：

```text
用毛泽东模式看我的内容增长。内容方向很多，平台很多，资源不够，我应该怎么集中兵力？
```

适合得到的回答方向：

- 不平均铺开
- 先抓主要阵地
- 先做根据地，再谈扩张
- 看群众基础，而不是只看自我表达

## 与“方法论工具箱”的区别

如果你同时使用 `qiushi-skill` 一类方法论工具箱：

- 这个仓库更像“人物视角，一体化回答”
- `qiushi-skill` 更像“方法模块，按需拼装”

简单说：

- 想问“毛泽东会怎么看”时，用这个
- 想问“先用矛盾分析还是调查研究还是持久战略”时，用方法论工具箱

## 边界与风险

这个 skill 有明确边界：

- 它只能基于公开资料提炼思维框架，不等于还原私人全部心境
- 它承认毛泽东晚年政治实践造成的高成本后果，不做洗白
- 它提供的是分析视角，不是历史定论
- 使用者应自行判断现实问题是否适合套用历史政治框架
